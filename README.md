# MVC
MVC (Model-View-Controller) — это паттерн проектирования, который разделяет приложение на три основных компонента: модель, представление и контроллер.

Тип паттерна: Кетер  Структурный

Принцип работы

1. Пользователь взаимодействует с представлением: 
Пользователь взаимодействует с представлением, то есть с интерфейсом.

2. Представление передает запрос контроллеру: 
Представление передает запрос на изменение или обновление данных контроллеру.

3. Контроллер обновляет модель: 
Контроллер обновляет модель, которая представляет состояние приложения.

4. Модель уведомляет представление об изменении: 
Модель уведомляет представление об изменении данных, которое затем обновляет пользовательский интерфейс.

Назначение: Деление кода на три модуля, упрощение тестирования (в сравнении с кодом в кнопках)

Метафора: Кафе, где:
Модель - кухня
Представление - зал,
Контроллер - администратор (видит заказы в зале и может чуть изменить работу кухни)

Проблемы: Модель, Контроллер и Представление друг с другом связаны, следовательно, сломалось что-то  = ищи по всей программе. Сложно вносить изменения.
